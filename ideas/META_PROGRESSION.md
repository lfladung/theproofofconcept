Meta Progression & Gear Evolution Design (Final)
Core Philosophy

Flat stats make you viable. Systems make you powerful.

This system is built on four pillars:

Amplification over accumulation — power comes from interaction, not stacking numbers
Commitment over randomness — progression is intentional and directed
Runs define power expression — builds come alive during gameplay
Progression is bounded — no infinite scaling or stat treadmill
System Overview

The progression model is composed of three layers:

Layer	System	Role
Permanent Identity	Gear Tiers	Defines what your build can do
Run Power	Infusions + Tempering	Defines what your build does this run
Bridge	Promotion	Converts performance → progression

Supporting systems:

Inscriptions (synergy acceleration)
Attunement (specialization)
Gems (expression)
Materials (directional economy)
Stats (bounded smoothing)
1. Gear Evolution (Primary Progression)
Structure

Each piece of gear evolves through 3 permanent tiers:

Tier 1 — Base
Neutral identity
Basic functionality
Minimal stats
Limited interactions
Tier 2 — Aligned
Commits to a pillar (Edge, Flow, Surge, etc.)
Unlocks core behavior
Moderate stat increase
Tier 3 — Specialized
Fully expresses the concept
Introduces:
New mechanics
Rule-breaking behavior
Final stat increase (capped)
Design Rules
Evolution is irreversible
No RNG outcomes
Each path is qualitatively different, not numerically superior
Example (Sword)
Tier 1 — Basic Blade
Standard melee, no specialization
Tier 2 — Edge Blade
Crit-focused behavior, overkill spill
Tier 3 — Execution Blade
Execute thresholds, primed target amplification
2. Run-Stepping System (Tempering)
Concept

During a run, gear can temporarily exceed its tier through Tempering States.

These are temporary amplifications, not permanent upgrades.

States
Tempered I (Early Run)
+5–10% stat boost
Minor behavior enhancement
Tempered II (Late Run)
+10–20% stat boost
Unlocks:
Partial Tier 3 mechanics
Enhanced synergy effects
Example

Tier 2 Edge Sword

Normal:
Crit bonuses
Overkill spill
Tempered I:
Extended crit window
Small execute threshold (preview)
Tempered II:
Partial execute mechanic
Stronger chaining
Purpose
Creates mid-run excitement
Gives “almost there” feeling
Encourages deeper runs
Lets players temporarily exceed limits
3. Promotion System (Run → Meta Bridge)
Concept

High-performance runs convert into permanent progression.

How It Works

During a run, players:

Reach Tempered II
Complete key objectives (bosses, depth, challenges)

This grants:

Promotion Progress toward the next tier
Result
Fill Promotion → Unlock next Tier upgrade
Design Impact
Progress tied to skill + execution
Avoids grind-only progression
Makes deep runs meaningful
4. Infusions (Run Identity Layer)
Role

Infusions define your build during a run.

They:

Unlock thresholds
Shape gameplay behavior
Enable synergy with gear
Interaction with Systems
Accelerate Tempering
Activate Inscriptions
Enhance Attuned gear
Amplify Gem effects
5. Inscriptions (Persistent Synergy)
Concept

Permanent upgrades tied to:

Gear
Pillar
Behavior

Only active when:

Matching infusion is present during a run
Example

Flow Inscription (Sword)

Tier I → +attack speed when Flow active
Tier II → earlier threshold activation
Tier III → reduced threshold requirements
Role
Accelerates builds
Enhances synergy
Does NOT provide power outside runs
6. Attunement (Specialization Layer)
Concept

Gear can be attuned toward a pillar.

Effects

Amplifies:

Infusion interactions
Gem effects
Pillar-specific mechanics
Example

Edge-attuned sword

Stronger crit scaling
Enhanced execution effects
Role
Long-term build direction
Encourages specialization
Works alongside evolution (not a replacement)
7. Gems (Expression Layer)
Concept

Gems modify behavior, not raw stats.

Examples
Edge → crits apply bleed
Flow → chaining extends dodge windows
Surge → charged attacks slow enemies
Model (Recommended: Fatigue System)
Gems weaken over repeated use
Eventually require replacement
Purpose
Encourage experimentation
Prevent solved loadouts
Keep builds fresh
8. Materials (Directional Economy)
Primary Resource: Pillar Materials

From runs:

Infusions → converted into materials

Examples:

Edge material
Flow material
Surge material
Usage
Gear evolution
Inscriptions
Attunement
Rules
No generic currency (no gold)
Materials are directional
Farming reinforces build intent
Secondary Resource: Resonant Dust (Optional)
Small universal resource
Used for pacing (rate limiting)
Does NOT influence decisions
9. Stat Progression (Bounded)
Purpose
Smooth difficulty curve
Provide sense of growth
Support late-game viability
Rules
Stats are:
Small
Capped
Non-dominant
Sources
Gear Tiers
Base damage
Survivability
Familiarity (Gear XP)
Level	Bonus
Familiar	+2%
Tempered	+4%
Masterwork	+6%
Target Scaling
~1.5x–2x total growth from start → endgame
10. Difficulty Scaling
Rule

Difficulty increases through complexity, not stats.

Methods
1. Enemy Composition
Multi-family encounters
Synergy-based threats
2. Behavior Complexity
Faster reactions
Smarter targeting
3. Density
More enemies
Layered pressure
4. Hybridization
Cross-pillar enemies
Example: Edge + Surge
Avoid
HP inflation
Pure damage scaling
11. Full Progression Loop
During Run
Fight enemies
Gain infusions
Reach thresholds
Enter Tempered states
Push toward deeper completion
Post Run Rewards
Pillar materials
Gems
Optional Resonant Dust
Between Runs

Player chooses to:

Evolve gear (identity)
Inscribe gear (synergy)
Attune gear (specialization)
Manage gems (expression)
Long-Term Outcome
Gear becomes personalized
Builds become intentional
Runs become deeper, not easier
